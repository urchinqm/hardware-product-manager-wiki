# Device Binding

## 1. 基本概念

Device Binding（设备绑定，将硬件设备与用户账号、App 或云端身份建立关联的流程）用于权限控制、远程管理和数据归属。

Ownership（所有权，表示哪个账号拥有设备管理权）和 Sharing（共享，允许其他账号使用部分权限）需要分开定义。

## 2. 工作原理

设备绑定通常通过设备标识、二维码、蓝牙、热点配网、局域网发现或云端激活码建立账号与设备关系。云端保存绑定关系，设备端保存绑定状态或令牌。

## 3. 核心参数

核心参数包括设备唯一 ID、绑定入口、绑定凭证、账号体系、权限模型、解绑规则、换绑规则、共享人数、离线策略和安全校验。

## 4. 参数影响

绑定凭证过弱会带来盗绑风险。解绑规则不清会造成二手设备无法使用。离线策略影响无网场景下的控制权限。

## 5. 优点

绑定机制能明确设备归属、支持远程控制、数据同步和售后服务。

## 6. 局限性

绑定流程涉及账号、设备、App、云端和网络，失败点多，用户感知强。

## 7. 产品经理关注点

必须定义首绑、解绑、换绑、共享、转让、恢复出厂后的绑定状态和设备被盗绑的处理。

## 8. 常见应用场景

常见于 IoT 设备、摄像头、车载设备、穿戴设备、智能家居和工业网关。

## 9. 需求文档需要明确内容

PRD 需要明确设备 ID、绑定流程、权限角色、解绑条件、换绑限制、异常提示、云端接口和隐私数据处理。

## 10. 异常状态设计

异常状态包括设备已被绑定、绑定超时、二维码失效、云端失败、设备离线、账号冲突、解绑失败和恢复出厂后状态不一致。

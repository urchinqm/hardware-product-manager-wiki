# Firmware Upgrade

## 1. 基本概念

Firmware Upgrade（固件升级，更新设备底层软件或嵌入式系统镜像的功能）用于修复缺陷、增加功能和更新安全能力。Firmware（固件，运行在硬件设备上的底层软件）通常比 App 更接近硬件控制。

## 2. 工作原理

设备获取升级包后进行版本校验、完整性校验、写入、切换和重启。Bootloader（引导加载程序，负责启动系统并可参与固件升级恢复的程序）常用于失败回滚和安全启动。

## 3. 核心参数

- 升级包大小
- 下载方式
- 校验算法
- 分区方案
- 回滚策略
- 最低电量或供电要求

## 4. 参数影响

升级包越大，下载时间和失败概率越高。校验不足会增加损坏包写入风险。无回滚策略会扩大升级失败影响。供电不稳定会增加设备变砖风险。

## 5. 优点

- 可持续修复问题和增加功能。
- 支持远程维护。
- 可降低召回和现场维修成本。

## 6. 局限性

- 失败会影响设备可用性。
- 需要服务端、App、设备端协同。
- 版本兼容和灰度策略复杂。

## 7. 产品经理关注点

Firmware Upgrade vs OTA 的关系是：OTA（Over-The-Air，空中升级）是一种远程升级方式，Firmware Upgrade 是升级对象和能力。需求中要区分升级对象、传输方式和发布策略。

## 8. 常见应用场景

- 修复固件缺陷
- 增加设备功能
- 安全补丁发布
- 工厂或售后刷机

## 9. 需求文档需要明确内容

- 版本命名规则
- 升级触发方式
- 下载和校验流程
- 断点续传要求
- 回滚策略
- 升级过程用户提示

## 10. 异常状态设计

- 下载失败
- 校验失败
- 存储空间不足
- 电量不足
- 写入失败
- 重启后新版本启动失败

# Factory Reset

## 1. 基本概念

Factory Reset（恢复出厂设置，将设备配置和用户数据恢复到出厂状态的功能）用于设备转让、维修、异常恢复和重新绑定。

## 2. 工作原理

设备执行恢复出厂设置时，会清除或重置指定数据分区、配置项、绑定关系和网络信息。不同产品必须区分 Reset Configuration（重置配置）和 Wipe User Data（清除用户数据）。

## 3. 核心参数

- 触发方式：按键、App、云端或本地命令。
- 清除范围：影响隐私、安全和售后。
- 执行时长：影响用户等待和超时设计。
- 断电保护：影响数据一致性。
- 恢复后状态：影响重新激活流程。

## 4. 参数影响

清除范围过小会残留隐私或绑定状态，清除范围过大会增加误操作损失。触发方式越多，越需要权限和防误触设计。断电保护不足会造成设备无法启动或状态不一致。

## 5. 优点

- 提供明确的异常恢复路径。
- 支持设备转让和重新绑定。
- 降低售后处理成本。

## 6. 局限性

- 误触发会造成用户数据丢失。
- 云端、本地和 App 状态需要一致。
- 执行中断需要专门处理。

## 7. 产品经理关注点

必须定义哪些数据清除、哪些数据保留。Factory Reset vs Reboot 的区别是前者改变持久化数据，后者只是重启系统。

## 8. 常见应用场景

- 用户换绑设备
- 二手转让
- 售后维修
- 配网失败后的恢复

## 9. 需求文档需要明确内容

- 触发入口和权限
- 二次确认方式
- 清除数据清单
- 云端解绑规则
- 执行进度和结果提示
- 失败后的恢复策略

## 10. 异常状态设计

- 用户误触发
- 执行中断电
- 清除部分成功
- 云端解绑失败
- 恢复后无法重新配网
- 设备卡在初始化状态

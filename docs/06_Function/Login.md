# Register/Login

## 1. 基本概念

Register/Login（注册/登录，用户创建账号并验证身份进入系统的流程）是设备管理、云端同步和权限控制的基础。

Authentication（认证，验证用户是谁）和 Authorization（授权，决定用户能做什么）需要区分。

## 2. 工作原理

用户通过手机号、邮箱、第三方账号或企业账号登录，服务端验证凭证后下发 Token（令牌，用于后续请求身份验证的凭证）。App 使用 Token 调用业务接口。

## 3. 核心参数

核心参数包括登录方式、验证码有效期、密码规则、Token 有效期、刷新机制、多端登录、注销账号、隐私同意和风控策略。

## 4. 参数影响

验证码有效期影响安全和体验。Token 有效期影响安全和免登录体验。多端策略影响家庭共享、售后和账号安全。

## 5. 优点

账号体系支持设备管理、数据同步、权限控制、售后追踪和跨设备体验。

## 6. 局限性

登录流程会增加使用门槛，网络异常、验证码失败和隐私合规会影响转化率。

## 7. 产品经理关注点

产品经理应判断哪些功能必须登录，哪些功能可游客使用。强制登录必须有清晰业务理由。

## 8. 常见应用场景

常见于 App 设备管理、云端数据、家庭共享、会员服务、售后工单和远程控制。

## 9. 需求文档需要明确内容

PRD 需要明确登录方式、注册流程、忘记密码、验证码、隐私协议、注销账号、多端登录、Token 过期和异常文案。

## 10. 异常状态设计

异常状态包括验证码错误、账号不存在、密码错误、Token 过期、账号冻结、网络失败、第三方登录失败和隐私协议未同意。

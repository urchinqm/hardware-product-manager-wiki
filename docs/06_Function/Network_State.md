# Network State

## 1. 基本概念

Network State（网络状态，设备或 App 当前网络连接、可用性和质量的综合状态）用于指导用户判断是否能远程控制、同步数据或升级。

Online（在线）不等于 Internet Reachable（可访问互联网），需要区分局域网连接和云端可达。

## 2. 工作原理

设备通过 WiFi、蜂窝、以太网或蓝牙接入网络，并通过心跳、DNS、云端连接、MQTT（Message Queuing Telemetry Transport，消息队列遥测传输协议）或 HTTP 请求判断网络状态。

## 3. 核心参数

核心参数包括连接类型、IP 状态、信号强度、云端连接、心跳周期、离线判定时间、重连策略、弱网阈值和状态展示。

## 4. 参数影响

心跳过短会增加功耗和流量，过长会导致离线发现慢。弱网阈值影响误报和漏报。状态定义不清会造成 App 与设备显示不一致。

## 5. 优点

清晰网络状态能降低用户困惑，提高远程控制、升级和告警体验。

## 6. 局限性

网络状态受路由器、运营商、云端、DNS、权限和设备功耗策略影响，单一指标不能代表全部体验。

## 7. 产品经理关注点

必须定义状态枚举：未配网、连接中、已连接无互联网、在线、离线、弱网、服务器不可用等。

## 8. 常见应用场景

常见于设备首页、远程控制、OTA、告警推送、视频连接、日志上传和售后诊断。

## 9. 需求文档需要明确内容

PRD 需要明确状态定义、检测方式、刷新周期、展示文案、重试策略、离线缓存、用户引导和日志字段。

## 10. 异常状态设计

异常状态包括无 IP、DNS 失败、云端断开、弱信号、路由器无互联网、SIM 欠费、服务器不可用和状态不同步。

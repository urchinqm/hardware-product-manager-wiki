# USB Connector

## 1. 基本概念

USB Connector（Universal Serial Bus Connector，通用串行总线连接器，用于承载 USB 数据、供电和设备识别的物理接口）是硬件产品中最常见的外部接口之一。

USB Connector 是机械和电气连接形态；USB Protocol（USB 协议，用于定义主机和设备之间数据通信规则）是通信协议。产品需求不能把两者混为一谈。

## 2. 工作原理

USB 连接器通过金属端子、外壳、焊脚和线缆连接主机与设备。设备插入后，主机通过供电、上拉/下拉识别、枚举和驱动加载建立通信。

连接器本体只提供物理连接，具体支持 USB 2.0、USB 3.x、充电或 USB PD（USB Power Delivery，USB 供电协议）取决于引脚、控制器、线缆和系统设计。

## 3. 核心参数

核心参数包括连接器类型、插拔寿命、额定电流、额定电压、接触电阻、端子数量、安装方式、焊接方式、外壳材质、屏蔽设计、防水等级和 ESD（Electrostatic Discharge，静电放电）保护。

常见形态包括 USB-A、USB-B、Micro-USB、Mini-USB 和 USB Type-C。

## 4. 参数影响

插拔寿命影响长期可靠性。额定电流影响充电和外设供电能力。安装方式影响结构强度和维修难度。屏蔽和接地影响 EMI（Electromagnetic Interference，电磁干扰）表现。

USB-A vs USB Type-C 的核心差异是 USB-A 有方向且主要用于传统主机侧，USB Type-C 可正反插并支持更复杂的角色和供电协商。

## 5. 优点

USB 生态成熟，用户认知强，适合供电、调试、数据传输、升级和外设扩展。

## 6. 局限性

USB 兼容性受连接器、线缆、控制器、协议、供电和操作系统共同影响，不能只靠接口外形判断能力。

## 7. 产品经理关注点

产品经理需要明确 USB 接口到底用于充电、数据、调试、外设、升级还是多用途复用。

如果产品对可靠性要求高，需要定义插拔次数、线缆规格、接口防护、防呆、进水处理和售后可维修性。

## 8. 常见应用场景

常见于手机、平板、车机、摄像头、调试板、传感器网关、工业设备、充电器、外接存储和量产治具。

## 9. 需求文档需要明确内容

PRD 需要明确接口形态、支持协议、充电能力、数据速率、线缆要求、插拔寿命、防水防尘、ESD 等级、结构位置、用户提示和异常处理。

不要只写“支持 USB”，要写清楚“USB Type-C，仅充电”或“USB Type-C，支持 USB 2.0 数据和 5V 供电”等边界。

## 10. 异常状态设计

异常状态包括无法识别设备、供电不足、接触不良、接口松动、线缆不兼容、进水、短路、过流、静电损伤和插拔后系统未恢复。

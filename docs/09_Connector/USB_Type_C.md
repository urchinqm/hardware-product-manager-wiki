# USB Type-C

## 1. 基本概念

USB Type-C（USB-C，可正反插的 USB 连接器标准形态）是一种物理连接器，不等于 USB 3.x 高速数据，也不等于快充。

USB PD（USB Power Delivery，USB 供电协议，用于在 USB Type-C 等接口上协商电压、电流和功率）是协议能力，需要芯片、线缆和电源共同支持。

## 2. 工作原理

USB Type-C 通过 CC（Configuration Channel，配置通道，用于检测插入方向、角色和供电能力）识别连接方向、主从角色和供电关系。

如果支持 USB PD，端口控制器会通过 CC 线进行供电能力协商，再决定电压、电流和角色切换。

## 3. 核心参数

核心参数包括端口角色、支持的 USB 数据版本、最大电流、是否支持 USB PD、是否支持 Alt Mode（Alternate Mode，替代模式，用于通过 Type-C 承载 DisplayPort 等非 USB 信号）、插拔寿命、防水等级、ESD 保护和线缆要求。

端口角色包括 Source（供电端）、Sink（受电端）和 DRP（Dual Role Power，供电和受电双角色）。

## 4. 参数影响

只接 USB 2.0 引脚的 Type-C 不能提供 USB 3.x 高速能力。不支持 PD 的 Type-C 不应宣传高功率快充。未正确设计 CC 电阻会导致设备无法被识别或供电异常。

USB Type-C vs Micro-USB 的核心差异是 Type-C 可正反插、可支持更高功率和更多角色，但设计复杂度更高。

## 5. 优点

USB Type-C 用户体验好，正反插，生态正在成为主流，适合统一充电、数据和调试接口。

## 6. 局限性

Type-C 外形相同但能力差异大，用户容易误解。线缆、充电器和设备能力不匹配会产生大量兼容性问题。

## 7. 产品经理关注点

产品经理必须把“接口形态”和“协议能力”分开写。Type-C 只能说明插头长相，不能自动说明速率、快充、视频输出或 OTG。

## 8. 常见应用场景

常见于手机、平板、笔记本、车机、摄像头、智能音箱、工业设备、开发板、充电底座和数据采集设备。

## 9. 需求文档需要明确内容

PRD 需要明确 Type-C 是否支持充电、数据、OTG、PD、视频输出、调试、固件升级、正反插检测、进水检测和异常提示。

需要写清楚支持的充电器、线缆、最大功率、协议边界和不支持功能。

## 10. 异常状态设计

异常状态包括 CC 检测失败、方向识别异常、PD 协商失败、线缆不支持、充电慢、无法传输数据、过流、过温、进水检测触发和角色切换失败。

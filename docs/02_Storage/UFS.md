# UFS

## 1. 基本概念

UFS（Universal Flash Storage，通用闪存存储，是一种面向高性能嵌入式设备的托管闪存标准）常用于高性能手机、车载设备和需要高速存储的智能硬件。

Full Duplex（全双工，读写可在两个方向同时进行的通信方式）和 Command Queue（命令队列，允许多个存储命令排队处理的机制）是 UFS 相比 eMMC 的重要架构优势。

## 2. 工作原理

UFS 使用高速串行链路连接主控和存储器，内部控制器管理 NAND Flash，并向系统提供块设备。

UFS 支持更高并发和更低延迟的访问模型，适合多任务系统和大吞吐场景。

## 3. 核心参数

核心参数包括容量、UFS 标准版本、Lane（通道）数量、顺序读写速度、随机读写性能、功耗状态、温度等级、寿命、封装和主控兼容性。

标准版本不等于实际产品性能，实际速度受主控、固件、温度、文件系统和测试条件影响。

## 4. 参数影响

顺序读写影响启动、安装、升级和媒体文件处理；随机性能影响应用响应和数据库；功耗状态影响待机；温度表现影响高负载持续写入。

UFS vs eMMC 的核心差异是接口架构、并发能力、性能上限和成本。

## 5. 优点

UFS 性能和并发能力通常优于 eMMC。

UFS 更适合高分辨率视频、高速日志、复杂应用和多任务系统。

## 6. 局限性

UFS 成本、平台要求和调试复杂度通常高于 eMMC。

低端 MCU 或简单 MPU 平台通常不会选择 UFS。

## 7. 产品经理关注点

产品经理应确认高性能存储是否是刚需，不能只因参数更高而选择 UFS。

如果产品涉及高清视频、AI 数据缓存或复杂应用，UFS 可能降低启动和写入瓶颈。

## 8. 常见应用场景

UFS 常用于智能手机、车机、高端摄像设备、AI 边缘终端和高性能 Android 设备。

## 9. 需求文档需要明确内容

PRD 需要明确容量、读写性能目标、持续写入场景、热降速行为、分区方案、文件系统、升级空间和日志策略。

如果涉及大文件，需求应定义单文件大小、持续写入时长、断电保护和写满行为。

## 10. 异常状态设计

UFS 异常状态包括链路初始化失败、读写超时、空间不足、热降速、分区损坏、升级失败和寿命告警。

产品应设计性能降级提示、日志保护、数据修复和恢复模式。

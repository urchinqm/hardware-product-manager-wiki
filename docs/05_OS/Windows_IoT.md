# Windows IoT

## 1. 基本概念

Windows IoT（Windows Internet of Things，微软面向嵌入式和物联网设备的 Windows 系统产品线）用于需要 Windows 生态、图形界面、外设兼容或企业管理能力的设备。

## 2. 工作原理

Windows IoT 基于 Windows 平台能力，提供设备驱动、应用运行、系统更新和安全管理能力。产品侧通常关注系统授权、硬件兼容、应用部署、启动体验和长期维护。

## 3. 核心参数

- 系统版本和授权方式
- 硬件平台兼容性
- 启动时间
- 更新策略
- 外设和驱动支持

## 4. 参数影响

系统版本影响生命周期和安全补丁。硬件兼容性决定驱动风险。启动时间影响设备可用体验。更新策略影响现场维护和停机风险。

## 5. 优点

- Windows 生态和外设兼容性强。
- 适合需要桌面应用或企业管理的设备。
- 开发和调试工具成熟。

## 6. 局限性

- 资源占用通常高于 RTOS 和轻量 Linux。
- 授权、版本和长期维护需要提前确认。
- 不适合极低功耗和强实时场景。

## 7. 产品经理关注点

Windows IoT vs Linux 的差异主要在生态、授权、资源占用、驱动模式和维护方式。Windows IoT vs RTOS 的差异在实时性、资源需求和应用复杂度。

## 8. 常见应用场景

- 工业 HMI
- 自助终端
- 医疗或商业显示设备
- 需要 Windows 应用生态的硬件

## 9. 需求文档需要明确内容

- 系统版本和授权方案
- 硬件规格边界
- 应用启动和退出规则
- 系统更新策略
- 外设清单
- 安全加固要求

## 10. 异常状态设计

- 系统启动失败
- 应用崩溃
- 驱动不可用
- 更新失败
- 授权异常
- 外设识别失败

# Linux

## 1. 基本概念

Linux（类 Unix 开源操作系统内核及其生态，在嵌入式产品中常指基于 Linux 内核构建的系统）适合网络、多进程、文件系统和复杂应用场景。

Kernel（内核，管理进程、内存、驱动和硬件资源的操作系统核心）是 Linux 系统的基础。

## 2. 工作原理

Linux 设备通常由 Bootloader 加载 Kernel，Kernel 挂载 Root Filesystem（根文件系统，系统启动后的主文件系统），再启动 systemd、init 或自定义进程管理器。

Driver（驱动程序，用于让操作系统控制硬件设备的软件）决定硬件能力能否被系统使用。

## 3. 核心参数

核心参数包括内核版本、启动时间、文件系统、驱动支持、内存占用、进程管理、网络协议、日志系统、升级机制和长期维护周期。

## 4. 参数影响

内核版本影响驱动、漏洞修复和维护周期。启动时间影响用户体验。文件系统影响掉电恢复和数据可靠性。日志策略影响问题定位和存储寿命。

Linux vs RTOS 的差异是 Linux 生态强、功能丰富，但实时性和资源占用通常不如 RTOS。

## 5. 优点

Linux 生态成熟，网络、文件系统、驱动、开源组件和开发工具丰富。

## 6. 局限性

Linux 系统复杂，启动时间、资源占用、安全维护和异常恢复需要专门设计。

## 7. 产品经理关注点

产品经理应确认 Linux 是为了网络、多媒体、应用生态还是硬件驱动，而不是默认选择。

## 8. 常见应用场景

常见于网关、摄像头、车载设备、工业控制器、NAS、机器人和带屏智能设备。

## 9. 需求文档需要明确内容

PRD 需要明确启动时间、关机策略、日志策略、升级策略、文件系统、网络协议、安全要求、恢复模式和长期维护周期。

## 10. 异常状态设计

异常状态包括 Kernel panic、进程崩溃、文件系统损坏、内存不足、磁盘满、驱动加载失败、网络服务异常和升级失败。

# Android

## 1. 基本概念

Android（基于 Linux 内核的移动和嵌入式操作系统平台）适合需要图形界面、应用框架、触控、多媒体和生态兼容的智能硬件。

AOSP（Android Open Source Project，Android 开源项目）是 Android 系统定制的基础来源。

## 2. 工作原理

Android 启动后由 Linux Kernel、HAL（Hardware Abstraction Layer，硬件抽象层）、系统服务、Framework（应用框架层）和 App（应用程序）共同提供功能。

## 3. 核心参数

核心参数包括 Android 版本、启动时间、Launcher、显示分辨率、触控、音视频能力、权限模型、OTA、应用安装方式、Google 服务需求和安全补丁周期。

## 4. 参数影响

Android 版本影响应用兼容、安全补丁和系统能力。系统裁剪影响启动时间和资源占用。权限策略影响蓝牙、定位、存储和后台行为。

Android vs Linux 的差异是 Android 提供完整应用框架和 UI 生态，普通 Linux 更灵活但应用框架需要自行构建。

## 5. 优点

Android UI、触控、多媒体、应用生态和开发工具成熟，适合带屏智能设备。

## 6. 局限性

Android 资源占用较高，系统定制、权限、认证、启动优化和长期维护成本不低。

## 7. 产品经理关注点

产品经理需要明确是否需要 App 生态、触控屏、多媒体和系统设置能力。无屏或低资源设备通常不应选 Android。

## 8. 常见应用场景

常见于车机、智能屏、商显、自助终端、平板类设备、机器人控制屏和多媒体终端。

## 9. 需求文档需要明确内容

PRD 需要明确 Android 版本、系统 UI、预装应用、权限策略、升级方式、恢复出厂、日志导出、开机动画和应用崩溃处理。

## 10. 异常状态设计

异常状态包括开机卡 Logo、系统服务崩溃、应用无响应、权限被拒、存储满、OTA 失败、触控异常和音视频服务异常。

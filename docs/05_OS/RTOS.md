# RTOS

## 1. 基本概念

RTOS（Real-Time Operating System，实时操作系统，用于在确定时间约束内调度任务和响应事件的操作系统）常用于 MCU 和实时控制设备。

Real-Time（实时性，系统在规定时间内完成响应的能力）不等于运行速度快，而是响应时间可预测。

## 2. 工作原理

RTOS 通过 Task（任务）、Scheduler（调度器）、Interrupt（中断）、Semaphore（信号量）、Queue（队列）和 Timer（定时器）组织并发逻辑。

## 3. 核心参数

核心参数包括任务数量、调度策略、Tick 周期、最大中断延迟、内存占用、栈大小、功耗模式、驱动支持和安全认证需求。

## 4. 参数影响

任务优先级影响实时响应。栈配置不足会导致崩溃。Tick 周期影响定时精度和功耗。中断延迟影响控制闭环和通信可靠性。

RTOS vs Linux 的差异是 RTOS 更适合确定性控制和资源受限场景，Linux 更适合复杂应用和生态。

## 5. 优点

RTOS 资源占用低、响应可预测，适合控制、采样、低功耗和实时通信。

## 6. 局限性

RTOS 生态和应用能力通常不如 Linux，复杂 UI、文件系统和网络服务需要额外组件。

## 7. 产品经理关注点

产品经理应明确实时性指标，例如最大响应时间、采样周期、控制周期和超时处理，而不是泛称“实时”。

## 8. 常见应用场景

常见于传感器节点、电机控制、电源管理、穿戴设备、工业控制、低功耗 IoT 和车载控制模块。

## 9. 需求文档需要明确内容

PRD 需要明确任务优先级、响应时间、采样周期、功耗状态、异常恢复、日志能力、升级策略和产测模式。

## 10. 异常状态设计

异常状态包括任务死锁、栈溢出、看门狗复位、中断丢失、队列溢出、内存不足、低功耗唤醒失败和升级中断。

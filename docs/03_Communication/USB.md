# USB

## 1. 基本概念

USB（Universal Serial Bus，通用串行总线，用于数据传输和供电的标准接口）常用于设备连接电脑、外设、存储、调试和充电。

USB Type-C（USB-C，可正反插的 USB 连接器形态）是连接器，不等同于具体传输速率或供电能力。

## 2. 工作原理

USB 由 Host（主机，发起枚举和通信控制的一方）管理 Device（设备，响应主机枚举和请求的一方）。设备接入后，主机通过 Enumeration（枚举，识别设备类型、配置和接口的过程）加载驱动并建立通信。

## 3. 核心参数

核心参数包括 USB 版本、连接器类型、Host/Device/OTG（On-The-Go，允许设备在主机和外设角色间切换）角色、供电电流、数据速率、线缆要求、驱动类型和 ESD（Electrostatic Discharge，静电放电）保护。

USB PD（USB Power Delivery，USB 供电协议，用于协商更高电压和功率）必须由端口控制器、线缆和电源共同支持。

## 4. 参数影响

USB 版本影响理论带宽，但实际速度还受控制器、线缆、驱动和存储性能影响。供电能力影响外设稳定性。角色定义错误会导致设备无法被识别。

USB-C vs USB 3.x 的差异是前者是接口形态，后者是数据传输规范。

## 5. 优点

USB 生态成熟，支持供电、数据、调试和外设扩展。

## 6. 局限性

USB 兼容性涉及线缆、操作系统、驱动、供电和协议，测试矩阵容易膨胀。

## 7. 产品经理关注点

必须明确 USB 是用于充电、数据传输、调试、固件升级、外接存储还是外设扩展。不要只写“支持 Type-C”。

## 8. 常见应用场景

常见于充电口、电脑数据导出、USB 摄像头、U 盘升级、ADB（Android Debug Bridge，Android 调试桥）调试、串口调试和外接网卡。

## 9. 需求文档需要明确内容

PRD 需要明确接口形态、USB 角色、数据速率、充电功率、文件传输模式、驱动需求、系统提示、线缆要求和异常提示。

## 10. 异常状态设计

异常状态包括无法枚举、供电不足、线缆不兼容、驱动缺失、文件传输中断、过流保护、接口进水腐蚀和角色切换失败。

# UART

## 1. 基本概念

UART（Universal Asynchronous Receiver/Transmitter，通用异步收发器，用于异步串行通信的硬件模块）常用于主控与模组、调试口、低速外设之间的数据通信。

TTL UART（Transistor-Transistor Logic UART，晶体管-晶体管逻辑电平串口）描述芯片侧电平；RS-232（Recommended Standard 232，传统串行通信电气标准）描述另一套电气层，不能直接混用。

## 2. 工作原理

UART 使用 TX（Transmit，发送线）、RX（Receive，接收线）和可选 GND（Ground，地线）传输数据。通信双方通过相同 Baud Rate（波特率，每秒符号数）约定数据速率，并按起始位、数据位、可选校验位和停止位解析数据。

## 3. 核心参数

核心参数包括波特率、数据位、停止位、Parity（校验位，用于简单错误检测）、电平标准、流控方式、线缆长度、接口保护和协议帧格式。

RTS/CTS（Request To Send / Clear To Send，请求发送/允许发送硬件流控）可降低接收端缓存溢出风险。

## 4. 参数影响

波特率越高，吞吐越高，但抗干扰和布线要求更高。电平不匹配会导致通信失败甚至损坏器件。无流控场景下，大数据连续发送可能丢包。

UART vs SPI 的差异是 UART 线少、异步、速度较低；SPI 线多、同步、吞吐更高。UART vs I2C 的差异是 UART 点对点更常见，I2C 更适合板内多从设备低速总线。

## 5. 优点

UART 简单、成本低、调试方便，适合日志、AT 指令和低速控制。

## 6. 局限性

UART 不适合高速大数据传输，也缺少内建寻址和复杂错误恢复能力。

## 7. 产品经理关注点

需要明确 UART 是调试接口、量产接口还是正式业务接口。正式业务接口必须定义协议帧、超时、重试、版本兼容和异常码。

## 8. 常见应用场景

常见于 4G/5G 模组 AT 指令、GPS 模组 NMEA 输出、蓝牙模组、工厂产测、设备调试日志和 MCU 与主控通信。

## 9. 需求文档需要明确内容

PRD 需要明确波特率、帧格式、电平、接口定义、协议命令、响应格式、超时、重试、日志开关和量产工具使用方式。

## 10. 异常状态设计

异常状态包括波特率不匹配、接线反接、电平不兼容、数据乱码、接收缓存溢出、命令无响应、通信超时和协议版本不一致。

# WiFi

## 1. 基本概念

WiFi（Wireless Fidelity，无线局域网通信技术，通常基于 IEEE 802.11 系列标准）用于设备接入局域网或互联网。SSID（Service Set Identifier，服务集标识，即无线网络名称）是用户看到的 WiFi 网络名。

## 2. 工作原理

设备通过扫描 SSID、选择网络、认证、关联、获取 IP 地址后接入网络。STA（Station，站点模式，设备作为客户端接入路由器）是智能硬件常见模式。

## 3. 核心参数

- 支持频段：常见为 2.4 GHz 和 5 GHz，具体以芯片规格为准。
- 支持安全模式：如 WPA2、WPA3，需按目标市场和系统能力确认。
- 天线设计：影响覆盖、吞吐和稳定性。
- 漫游能力：影响移动场景下网络切换体验。
- 配网方式：影响首连成功率和用户体验。

## 4. 参数影响

频段影响覆盖范围、干扰和速率。安全模式影响兼容性和安全合规。天线布局不良会造成弱信号、吞吐下降和断连。配网方式会直接影响新用户激活成功率。

## 5. 优点

- 吞吐能力通常高于低功耗近场无线。
- 适合视频、日志上传和固件下载。
- 可直接接入互联网服务。

## 6. 局限性

- 功耗高于 BLE。
- 首次配网流程容易受路由器、手机系统和权限影响。
- 弱网和漫游场景需要专门设计。

## 7. 产品经理关注点

需要明确 STA Mode（站点模式）和 AP Mode（接入点模式）的使用边界。还要定义配网入口、失败提示、弱网策略、断网缓存、重连策略和安全要求。

## 8. 常见应用场景

- 摄像头联网
- 车机或网关联网
- 固件下载
- 大日志上传

## 9. 需求文档需要明确内容

- 支持频段和安全模式
- 配网方式和步骤
- IP 获取方式
- 断网、弱网和重连策略
- 网络状态展示
- 隐藏 SSID 和密码错误处理

## 10. 异常状态设计

- 扫描不到网络
- 密码错误
- DHCP 获取 IP 失败
- 已连接但无法访问互联网
- 弱信号频繁断连
- 路由器安全模式不兼容

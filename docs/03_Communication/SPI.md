# SPI

## 1. 基本概念

SPI（Serial Peripheral Interface，串行外设接口，是一种同步串行通信总线）常用于主控与 Flash、显示屏、传感器和高速外设之间通信。

Master（主设备，发起时钟和片选控制的设备）和 Slave/Peripheral（从设备/外设，响应主设备访问的设备）是 SPI 典型角色。

## 2. 工作原理

SPI 通常使用 SCLK（Serial Clock，串行时钟）、MOSI（Master Out Slave In，主出从入）、MISO（Master In Slave Out，主入从出）和 CS（Chip Select，片选）信号。主设备拉低片选后，通过时钟同步发送和接收数据。

## 3. 核心参数

核心参数包括时钟频率、CPOL（Clock Polarity，时钟极性）、CPHA（Clock Phase，时钟相位）、数据位宽、片选数量、总线长度、电平、读写命令和 DMA（Direct Memory Access，直接内存访问）支持。

SPI Mode（SPI 模式，由 CPOL 和 CPHA 组合定义）必须与外设规格书一致。

## 4. 参数影响

时钟频率影响吞吐和信号完整性。片选数量影响可挂载外设数量。模式不匹配会导致读写错误。板级走线过长或干扰严重会导致高速通信不稳定。

SPI vs I2C 的差异是 SPI 更快、线更多、无标准寻址；I2C 线少、有地址、速度通常较低。

## 5. 优点

SPI 吞吐高、协议简单、全双工能力好，适合板内高速外设。

## 6. 局限性

SPI 占用引脚多，缺少统一设备发现和标准错误恢复机制。

## 7. 产品经理关注点

需要确认 SPI 外设是否影响启动、显示刷新、采样率或存储速度。高频 SPI 需求必须让硬件评估走线、阻抗、EMI（Electromagnetic Interference，电磁干扰）和时序裕量。

## 8. 常见应用场景

常见于 SPI NOR Flash、LCD 显示屏、触控芯片、ADC、传感器、无线模组和安全芯片。

## 9. 需求文档需要明确内容

PRD 需要明确外设清单、速度目标、数据量、刷新率、启动依赖、错误处理、总线占用冲突和调试接口。

## 10. 异常状态设计

异常状态包括片选错误、模式错误、读写超时、设备无响应、数据校验失败、总线冲突、启动读取失败和高频信号不稳定。

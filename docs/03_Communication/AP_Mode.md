# AP Mode

## 1. 基本概念

AP Mode（Access Point Mode，接入点模式，设备自身创建 WiFi 热点供手机或其他设备连接）常用于智能硬件首次配置、无路由器直连和本地维护。

## 2. 工作原理

设备启动 SoftAP（Software Access Point，软件接入点，由设备芯片或系统创建的无线热点）后，手机连接该热点，再通过本地 HTTP、TCP、UDP 或私有协议写入配网信息或读取状态。

## 3. 核心参数

- SSID 命名规则：影响用户识别设备。
- 热点密码策略：影响安全性和连接便利性。
- AP 存活时间：影响配网窗口和安全暴露时间。
- 本地通信协议：影响 App 和设备联调复杂度。
- 设备最大连接数：影响维护和多人操作场景。

## 4. 参数影响

SSID 规则不清晰会增加用户选错热点的概率。无密码热点降低操作门槛但增加安全风险。AP 存活时间过长会增加误连和攻击面，过短会降低配网成功率。

## 5. 优点

- 不依赖外部路由器即可建立本地连接。
- 适合首次配网和现场维护。
- 流程可控，便于 App 引导。

## 6. 局限性

- 手机连接设备热点后可能临时失去互联网。
- 不同手机系统对无互联网 WiFi 的处理不同。
- 热点安全和超时退出必须设计。

## 7. 产品经理关注点

AP Mode vs Hotspot 的区别是角色不同：AP Mode 指设备创建接入点，Hotspot 通常指手机或设备共享网络。需求中必须写清楚谁创建热点、谁连接热点、是否需要互联网。

## 8. 常见应用场景

- IoT 设备首次配网
- 设备本地调试
- 无云端场景的数据读取
- 工厂产测配置

## 9. 需求文档需要明确内容

- AP 触发条件
- SSID 和密码规则
- AP 超时退出策略
- App 连接引导
- 本地协议和端口
- 配网成功后的 AP 关闭规则

## 10. 异常状态设计

- 手机无法发现 AP
- 手机连接 AP 后自动切回移动网络
- 本地接口超时
- 配网信息写入失败
- AP 长时间未关闭
- 多手机同时连接造成状态冲突

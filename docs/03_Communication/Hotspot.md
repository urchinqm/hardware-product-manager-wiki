# Hotspot

## 1. 基本概念

Hotspot（热点，通过无线方式共享网络连接的功能）通常指手机、车机或网关创建可供其他设备接入的 WiFi 网络。Tethering（网络共享，把一个设备的蜂窝或有线网络共享给其他设备）是热点常见用途。

## 2. 工作原理

热点设备创建无线接入点，并通过 NAT（Network Address Translation，网络地址转换）或桥接方式让接入设备访问上游网络。接入设备通常把热点当作普通 WiFi 使用。

## 3. 核心参数

- 热点名称和密码规则
- 支持频段
- 最大接入设备数
- 上游网络类型
- 流量和功耗策略

## 4. 参数影响

最大接入设备数影响多人共享体验。上游网络质量决定实际互联网体验。热点持续开启会增加功耗和发热。弱密码会增加被蹭网和安全风险。

## 5. 优点

- 可在无固定路由器场景提供联网能力。
- 用户理解成本低。
- 适合移动场景临时联网。

## 6. 局限性

- 依赖上游网络质量。
- 功耗和发热压力较高。
- 需要处理流量限制和安全提示。

## 7. 产品经理关注点

Hotspot vs AP Mode 的区别是热点强调共享上游网络，AP Mode 强调设备作为本地接入点。需求中不能混用这两个概念。

## 8. 常见应用场景

- 手机给设备临时联网
- 车机共享蜂窝网络
- 网关给从设备提供局域网
- 现场维护临时网络

## 9. 需求文档需要明确内容

- 热点创建入口
- 名称和密码生成规则
- 最大连接数
- 流量限制提示
- 关闭策略
- 安全和权限提示

## 10. 异常状态设计

- 热点开启失败
- 密码错误
- 接入设备过多
- 上游网络不可用
- 流量耗尽
- 热点被系统关闭

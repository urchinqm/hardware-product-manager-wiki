# Bluetooth Classic

## 1. 基本概念

Bluetooth Classic（经典蓝牙，用于持续连接、语音和较高吞吐外设通信的蓝牙技术）适合音频、串口透传和车载外设连接。BR/EDR（Basic Rate / Enhanced Data Rate，基本速率/增强数据速率）是 Bluetooth Classic 的核心物理层和链路能力名称。

## 2. 工作原理

Bluetooth Classic 通过配对、认证、链路建立和 Profile（配置文件，定义设备间功能交互方式）完成通信。产品侧通常不直接设计底层射频协议，而是定义设备角色、配对流程、连接保持策略和断连恢复策略。

## 3. 核心参数

- 蓝牙版本：以芯片规格书和认证资料为准。
- 支持 Profile：如 A2DP、HFP、SPP，必须按产品功能确认。
- 连接数量：影响是否支持多设备同时连接。
- 发射功率和接收灵敏度：影响连接距离和抗干扰能力。
- 功耗：影响待机、连接保持和持续传输续航。

## 4. 参数影响

支持的 Profile 决定产品能否实现音频、通话或数据透传。连接数量决定多设备体验。发射功率提高通常有利于距离，但会增加功耗和射频合规压力。功耗策略会影响连接稳定性和续航。

## 5. 优点

- 生态成熟，手机、车机、耳机和外设支持广。
- 适合持续连接和音频类业务。
- Profile 体系清晰，便于定义互操作需求。

## 6. 局限性

- 相比 BLE，低功耗周期广播和小数据低功耗场景不占优势。
- 配对和 Profile 兼容性需要实机验证。
- 多设备连接会增加状态管理复杂度。

## 7. 产品经理关注点

需要明确目标设备、连接流程、配对入口、连接优先级、多设备策略、断连恢复和兼容性测试范围。Bluetooth Classic vs BLE 的选择应基于数据量、功耗、连接持续性和系统支持能力。

## 8. 常见应用场景

- 蓝牙音频播放
- 车载免提通话
- 串口透传设备
- 遥控器或传统外设连接

## 9. 需求文档需要明确内容

- 支持的 Profile 清单
- 配对和解绑流程
- 连接状态展示
- 多设备连接规则
- 断连、重连和超时策略
- 兼容性测试设备清单

## 10. 异常状态设计

- 配对失败
- 认证失败
- 已连接但业务 Profile 不可用
- 连接中断
- 多设备抢占连接
- 系统蓝牙关闭

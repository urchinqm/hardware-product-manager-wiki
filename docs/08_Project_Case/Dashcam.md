# Dashcam

## 1. 基本概念

Dashcam（行车记录仪，用于车辆行驶过程中采集视频、音频、位置和事件数据的设备）核心价值是证据记录和行车安全辅助。

Loop Recording（循环录像，存储满后按规则覆盖旧视频的录像机制）是行车记录仪的基础功能。

## 2. 工作原理

设备通过摄像头、ISP、编码器、存储和电源管理持续录制视频，并通过 G-Sensor、GPS/GNSS 和按键触发事件锁定。

## 3. 核心参数

核心参数包括分辨率、帧率、码率、视场角、夜视能力、存储卡规格、循环录像时长、碰撞灵敏度、启动时间、工作温度和供电方式。

## 4. 参数影响

码率和分辨率影响画质和存储占用。存储卡速度影响丢帧风险。碰撞阈值影响误锁和漏锁。高温影响稳定性和寿命。

## 5. 优点

行车记录仪能持续记录道路事件，支持事故取证、停车监控和驾驶行为分析。

## 6. 局限性

录像质量受光线、镜头、安装角度、玻璃反光、存储卡质量和供电影响。

## 7. 产品经理关注点

产品经理必须定义录像质量、文件策略、异常提示、证据保护和用户取证路径。

## 8. 常见应用场景

常见于私家车、商用车、摩托车、网约车、车队管理和停车监控。

## 9. 需求文档需要明确内容

PRD 需要明确录像规格、文件命名、循环覆盖、锁定文件、存储卡要求、碰撞触发、停车监控、时间同步和导出方式。

## 10. 异常状态设计

异常状态包括未插卡、卡速不足、录像失败、文件损坏、镜头遮挡、过温关机、时间错误、GPS 无定位和事故文件未锁定。

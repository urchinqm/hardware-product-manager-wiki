# Motorcycle

## 1. 基本概念

Motorcycle Smart Device（摩托车智能设备，面向摩托车场景的导航、记录、通信或车况监控硬件）需要考虑震动、防水、强光和骑行安全。

IP Rating（Ingress Protection Rating，防尘防水等级）是摩托车设备结构要求中的关键指标。

## 2. 工作原理

摩托车设备通常集成显示、GNSS、蓝牙、传感器、供电转换和车载通信，通过车把安装或车身安装提供导航、记录和状态显示。

## 3. 核心参数

核心参数包括屏幕亮度、防水等级、抗震能力、工作温度、供电范围、GNSS 性能、蓝牙连接、按键手套可用性和安装方式。

## 4. 参数影响

屏幕亮度影响阳光下可读性。防水和抗震影响可靠性。供电范围影响不同车型兼容。按键和 UI 影响骑行中可操作性。

## 5. 优点

摩托车智能设备能提升导航、记录、安全提醒和车辆状态感知。

## 6. 局限性

摩托车环境比车内更严苛，雨水、震动、强光、温差和供电波动都会影响产品稳定性。

## 7. 产品经理关注点

产品经理应优先考虑骑行安全，限制复杂交互，保证关键信息一眼可读。

## 8. 常见应用场景

常见于摩托车导航屏、骑行记录仪、胎压监控显示、车队通信终端和防盗定位设备。

## 9. 需求文档需要明确内容

PRD 需要明确安装方式、防水等级、抗震要求、屏幕可视性、手套操作、供电保护、蓝牙连接、定位和异常提示。

## 10. 异常状态设计

异常状态包括进水、震动导致接触不良、过温、低温启动失败、供电波动、蓝牙断连、定位失败和安装松动。

# SoC

## 1. 基本概念

[COMMON] SoC（System on Chip，片上系统，把 CPU、GPU、NPU、存储控制器、接口控制器和多媒体模块等系统级功能集成到单芯片的方案）常作为复杂智能硬件的主平台。

[COMMON] NPU（Neural Processing Unit，神经网络处理单元，用于加速机器学习推理计算）是否存在和性能如何，取决于具体 SoC 型号。

## 2. 工作原理

[COMMON] SoC 通过片上总线连接 CPU、GPU（Graphics Processing Unit，图形处理器）、ISP（Image Signal Processor，图像信号处理器）、存储控制器、显示控制器和外设控制器。

[COMMON] 产品启动时，SoC 需要完成电源时序、时钟初始化、内存训练、Bootloader 加载、内核启动和应用启动。

## 3. 核心参数

[COMMON] SoC 核心参数包括 CPU 架构和核心数、GPU 能力、NPU 算力、视频编解码、ISP 能力、显示输出、摄像头输入、内存支持、存储接口、网络接口、功耗、散热、封装和生命周期。

[COMMON] 软件参数包括 SDK、BSP、驱动完整性、Android/Linux 版本、OTA 支持、安全启动和量产工具。

## 4. 参数影响

[COMMON] CPU 和内存影响系统响应；GPU 和显示接口影响 UI 流畅度；ISP 影响图像质量；视频编解码影响录像和播放；NPU 影响端侧 AI 功能；功耗和散热影响结构设计和稳定性。

[COMMON] SoC vs MCU 的差异是系统能力和复杂度；SoC vs MPU 的差异是 SoC 更强调系统级模块集成，而 MPU 更强调处理器本体。

## 5. 优点

[COMMON] SoC 集成度高，适合复杂产品平台化。

[COMMON] SoC 可减少外围芯片数量，并提供多媒体、AI、显示和高速接口能力。

## 6. 局限性

[COMMON] SoC 选型锁定后，软件生态、驱动质量和供应链会深度影响项目成败。

[COMMON] SoC 项目调试复杂，常涉及硬件、BSP、驱动、系统、应用和云端多团队协作。

## 7. 产品经理关注点

[COMMON] 产品经理必须把 SoC 选型和目标功能绑定，而不是只看峰值算力或宣传参数。

[COMMON] 需要重点确认供应商支持能力、参考设计成熟度、量产案例、长期供货和认证资料。

## 8. 常见应用场景

[COMMON] SoC 常用于智能摄像头、行车记录仪、车机、机器人、网关、智能屏、电视盒子和边缘 AI 设备。

## 9. 需求文档需要明确内容

[COMMON] PRD 需要明确性能目标、启动时间、屏幕规格、摄像头规格、视频规格、AI 功能、功耗上限、散热约束、接口清单、升级策略和安全要求。

[COMMON] 涉及 AI 或视频功能时，需求必须明确输入格式、输出指标、帧率、延迟和失败降级策略。

## 10. 异常状态设计

[COMMON] SoC 异常状态包括高温降频、内存不足、视频编码失败、摄像头初始化失败、显示异常、外设驱动异常、启动卡住和系统升级失败。

[COMMON] 对用户可感知异常，应定义日志采集、错误码、自动恢复、用户提示和售后导出方式。

# Compass

## 1. 基本概念

Compass（电子罗盘，用于输出设备朝向的传感器功能）通常依赖 Magnetometer（磁力计，用于测量磁场强度和方向的传感器）计算方位角。

## 2. 工作原理

电子罗盘通过测量地磁场方向，并结合姿态补偿算法输出航向。实际产品中常与 Accelerometer（加速度计，用于测量线性加速度的传感器）和 Gyroscope（陀螺仪，用于测量角速度的传感器）融合。

## 3. 核心参数

- 方位角精度：影响导航方向可信度。
- 校准方式：影响用户操作和长期稳定性。
- 抗磁干扰能力：影响复杂环境表现。
- 输出频率：影响方向刷新速度。
- 坐标系定义：影响软件集成。

## 4. 参数影响

校准不足会造成方向偏移。磁干扰会导致航向跳变。坐标系定义不清会造成前后左右方向错误。

## 5. 优点

- 可提供绝对方向参考。
- 适合地图朝向和导航辅助。
- 与 IMU 融合后可改善姿态体验。

## 6. 局限性

- 容易受磁铁、电机、电流和金属结构影响。
- 室内和车内环境需要重点验证。
- 校准流程会影响用户体验。

## 7. 产品经理关注点

Compass vs Magnetometer 的区别是功能输出和底层传感器不同：Compass 是方向功能，Magnetometer 是磁场测量器件。需求中应分别定义用户结果和底层数据。

## 8. 常见应用场景

- 地图朝向
- 户外导航
- 设备安装方向判断
- 姿态融合

## 9. 需求文档需要明确内容

- 方位角输出定义
- 校准入口和提示
- 干扰提示
- 坐标系和安装方向
- 与 GPS/GNSS 的融合策略
- 异常状态展示

## 10. 异常状态设计

- 需要校准
- 磁干扰过强
- 方位角跳变
- 传感器不可用
- 安装方向配置错误
- 融合算法输出不稳定

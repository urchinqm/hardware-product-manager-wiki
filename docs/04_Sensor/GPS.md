# GPS

## 1. 基本概念

GPS（Global Positioning System，全球定位系统，是美国建设的卫星导航定位系统）用于提供位置、速度和时间信息。

GNSS（Global Navigation Satellite System，全球导航卫星系统，是多国卫星导航系统的统称）包含 GPS、北斗、GLONASS 和 Galileo 等系统。

## 2. 工作原理

GPS 接收机接收卫星信号，通过卫星时间和轨道信息计算伪距，再用多颗卫星解算位置。冷启动、热启动和辅助定位会影响首次定位时间。

A-GPS（Assisted GPS，辅助 GPS，通过网络提供星历或粗略位置以加快定位）常用于手机和联网设备。

## 3. 核心参数

核心参数包括首次定位时间、定位精度、刷新率、灵敏度、支持星座、天线类型、功耗、冷启动条件、热启动条件和弱信号表现。

NMEA（National Marine Electronics Association，常见定位数据输出格式）常用于模块向主控输出经纬度、速度和时间。

## 4. 参数影响

天线位置影响信号质量。刷新率影响轨迹平滑和功耗。弱信号环境会导致漂移、丢星和首次定位变慢。室内定位能力不能按户外指标承诺。

GPS vs GNSS 的差异是 GPS 是具体系统，GNSS 是统称；产品需求应写清支持哪些星座。

## 5. 优点

GPS 覆盖广，适合户外定位、轨迹记录和时间同步。

## 6. 局限性

GPS 对遮挡、室内、隧道、高楼峡谷和天线设计敏感。

## 7. 产品经理关注点

产品经理应定义定位场景：户外、车内、室内、运动、静止、弱网或无网。不同场景的定位体验差异很大。

## 8. 常见应用场景

常见于行车记录仪、运动设备、车载终端、资产追踪、无人机、户外设备和时间同步。

## 9. 需求文档需要明确内容

PRD 需要明确支持星座、定位精度目标、首次定位时间、刷新率、轨迹保存、无信号提示、时间同步和隐私授权。

## 10. 异常状态设计

异常状态包括无卫星、弱信号、定位漂移、速度异常、时间未同步、天线断开、模块无响应和权限未授权。
